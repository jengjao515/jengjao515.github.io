Welcome to Scope City global, local, and everything in between date posted 04/23/2015 16:44

Ruby has a variety of variable types. But let's start with the basic stuff.

Every variable has a scope type: global or local.

A local variable can only be accessed within where it was declared. If it was created within a method or a class, it can only be used within those areas. Calling the variable outside will output an error in the terminal.

A global variable can be accessed anywhere after the variable has been declared. You can reference to past global variables declared in past code, but you can't reference to new variables down the code.

gvariable = "I am Groot!"

def spaceship
      p "Hello!" + gvariable
      lvariable = "I am Vin Diesel."
end

spaceship => "Hello! I am Groot!"
p lvariable => This will not show "I am Vin Diesel." You will get an error.

Now you know that Groot and Vin Diesel are in separate places. Groot is the best, so he can show up whenever you need him to. But Vin Diesel's a busy guy so he's limited to when he can make an appearance.
