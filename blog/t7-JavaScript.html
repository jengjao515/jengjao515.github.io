Javascript needs friends Ruby hashes vs. Javascript Objects date posted 04/30/2015 13:29

The first thing you learn in Ruby is that everything is an object. The first thing you learn in Javascript is that you need to code everything out. Although the logic behind the languages are similar, Ruby's built-in methods make the code look a lot cleaner. Javascript also has built-in methods, however, they take a little more space on the screen to use.

So let's get to our real discussion about Ruby hashes vs. Javascript objects. They are typically created the same way.

Javascript object
var javascript = {}

Ruby hash
ruby = {}
# or
ruby = Hash.new

Now let's add some fat to these hashes/objects.

Javascript object
var javascript = {
  name: "Javascript",
  nickname: "JS"
}

Ruby hash
ruby = {
  name => "Ruby",
  nickname => "Rb" }

It's important to note that Javascript object pairs are called properties and values. In Ruby, they're called keys and values. So how do we add items later on? These are pretty similar, as well. You can also change the values of existing properties through the same way as adding.

Javascript object
var javascript = {
  name: "Javascript",
  nickname: "JS"
}
javascript.quote = "Add me!"
// or
javascript["quote"]

Ruby hash
ruby = {
  name => "Ruby",
  nickname => "Rb" }
ruby[quote] = "Add me, too!"

Both hashes and objects store information that can be accessed and changed. They're pretty useful when you want to categorize some specific information, for example, a phonebook.
